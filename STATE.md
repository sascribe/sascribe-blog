## Session 10 Changes (2026-04-17)

### Countdown Interstitial Flow
- Timer changed from 5s → **8 seconds**
- Countdown completion now redirects to **offer URL** (not homepage)
- "No Thank You" button goes to **offer URL** (not homepage)
- Homepage email form submission sets `localStorage qrp_lead_captured=true` + cookie (30-day)
- Offer CTAs check lead_captured flag — if set, skip interstitial entirely
- Timer **pauses on input focus**, resumes after 3s inactivity
- Form submit inside interstitial cancels timer and goes directly to offer
- Never sends user to homepage from interstitial flow

### QR Code Auto-Generation (T9+)
- Minimal pure-JS QR code generator implemented in worker (no external deps)
- Encodes `https://qr-perks.com/{truck_id}` as inline SVG
- On first generation, POSTs SVG to `/api/save-qr-code` → persists in `trucks.qr_code_svg`
- Subsequent loads read from Supabase
- Four download buttons: **SVG, PDF, JPEG, PNG** (blob URL pattern)
- Filenames: `QR-Perks-T{n}.{ext}`
- EN/ES scan-and-test notice below every QR code

### Email Unsubscribe
- Unsubscribe link in every email footer includes recipient email: `/unsubscribe?email={encoded}`
- `/unsubscribe?email=...` handler flags `email_captures.status = 'unsubscribed'`
- Shows bilingual (EN/ES) confirmation page
- `sendEmail()` checks unsubscribed status before sending to subscriber templates
- `handleCapture` checks unsubscribed status before re-adding and sending welcome email

### SMS Legal Compliance
- `/api/sms-webhook` route handles STOP/CANCEL/UNSUBSCRIBE/END/QUIT replies
- Flags `email_captures.sms_unsubscribed = true` on opt-out
- Returns TwiML response with confirmation message
- Schema columns added: `sms_unsubscribed BOOLEAN`, `sms_unsubscribed_at TIMESTAMPTZ`

### Welcome Email Rewrite
- Subject: "You're In ✅ — QR Perks" (kept checkmark emoji)
- Body: insider list messaging, deals come to them directly, no truck scanning mention
- Bilingual: EN and ES based on `lang` column in `email_captures`
- Unsubscribe link in footer with encoded email

### Email Routing (Cloudflare)
- Email Routing **enabled** on qr-perks.com zone via API
- DNS records auto-synced by Cloudflare
- ⚠️ **Requires manual step**: Log into CF dashboard → qr-perks.com → Email Routing → Add destination `qrperks@gmail.com` → verify via Gmail → create rules for `support@`, `privacy@`, `contact@`

### Privacy Policy Fix
- Removed duplicate standalone `privacy@qr-perks.com` paragraph (now appears exactly once in "Your Rights" section)

### Logo Links
- All instances of "QR PERKS" logo/name in header/nav are now `<a href="/">` links:
  - Landing page header
  - Auth shell (login/signup/forgot/reset)
  - Driver dashboard nav (QRP)
  - Admin nav (QRP ADMIN → /admin/dashboard)
  - Legal page shell
  - Contact page

### Contact Form Fix
- `handleContactPost` now uses `env.RESEND_API_KEY` and properly fires Resend API
- Returns correct `{ ok: false, error: "..." }` on failure
- Client-side JS checks `d.ok` before showing confirmation
- Sends to `support@qr-perks.com` with `reply_to` set to submitter's email
